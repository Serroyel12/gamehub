#  RankArena - Proyecto El Hexágono

**RankArena** es una plataforma competitiva y social diseñada para gamificar la experiencia de usuario. Este proyecto se desarrolla bajo el marco de trabajo **Scrum**, priorizando la entrega de valor constante y la excelencia técnica.

---

##  Tralla Técnica y Buenas Prácticas
Para asegurar la calidad del producto final, hemos aplicado las siguientes disciplinas de ingeniería:

* **Control de Versiones**: Uso intensivo de **Git**, manteniendo un historial de aproximadamente **30 commits por versión** para garantizar trazabilidad y control total de cambios.
* **Refactorización**: Ciclos constantes de mejora de código para eliminar deuda técnica y asegurar la mantenibilidad.
* **Testing**: Validación de funcionalidades para asegurar que cada incremento cumple con el **DoD (Definition of Done)** antes de su integración.
* **Gestión con Jira**: Organización integral del Backlog, donde cada Historia de Usuario incluye tareas de diseño, implementación y aceptación.

---

##  Metodología Ágil Aplicada
* **Priorización MoSCoW**: Clasificación de tareas en *Must, Should, Could* y *Won't* para asegurar la entrega de requisitos críticos.
* **Estimación Relativa**: Uso de **Puntos de Historia** en lugar de horas para medir el esfuerzo real del equipo.
* **Gestión de Riesgos**: Se ha reservado un **20% del tiempo** para imprevistos, integración y refactorización técnica.
* **Retrospectiva Estrella de Mar**: Análisis post-sprint centrado en:
  * **Keep Doing**: Prácticas exitosas.
  * **Start Doing**: Nuevas propuestas de mejora.
  * **Stop Doing**: Eliminación de tareas que no aportan valor.
* **Motivación**: Implementación de **Kudos** para reconocer el desempeño técnico y fomentar el buen ambiente grupal.

---

##  Equipo: El Hexágono
Somos un equipo de 7 integrantes comprometidos con la ingeniería de software de calidad.
* **Metodología**: Scrum Core con roles definidos y rotativos.
* **Comunicación**: Gestión mediante Daily Meetings, Sprint Planning y Demos al cliente.

---

> **Nota Técnica**: El desarrollo se ha centrado en que las herramientas trabajen a favor del equipo, utilizando el sistema de versiones no solo como repositorio, sino como un mecanismo de seguridad y mejora continua.
